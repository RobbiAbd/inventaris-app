# Checkpoint — 2 Juni 2026

## Ringkasan

Hari ini fokus menyelesaikan fitur inti inventaris, perbaikan UI/UX, otorisasi per role, riwayat aktivitas, dashboard HRD dengan chart, serta push pertama ke GitHub.

---

## Fitur yang diselesaikan

### Master Data & Pengaturan
- Master data dinamis (`master_options`) — halaman `/pengaturan/master-data`
- Field **Batasi jenis vendor** diperjelas label + help text; opsi default "Tidak dibatasi (semua vendor)"
- Modal Master Data diperbesar + scroll agar tombol tidak hilang (slot native `UModal` `#body` / `#footer`)

### Modul & CRUD
- Modul **Pengguna** (HRD): CRUD user dengan role HRD / KARYAWAN
- **Riwayat aktivitas** (`activity_logs`): halaman `/riwayat`, timeline di detail barang
- **Soft delete** untuk entitas utama
- Perbaikan **edit barang** dan scroll layout

### Auth & Role
- Karyawan hanya melihat **barang yang ditugaskan** ke user login (`barangScopeForUser`, `assertBarangAccess`)
- Menu **Vendor** disembunyikan dari karyawan (sidebar + API GET hanya HRD)
- Middleware `hrd` dan auth global diperkuat (async)

### Dashboard HRD
- Chart agregat via `GET /api/dashboard/stats` + Chart.js / vue-chartjs:
  - Donut: status barang, pengguna per role
  - Bar: kategori barang, penugasan barang
  - Pie: tipe perolehan, vendor per jenis
- Karyawan: kartu statistik barang sendiri saja (tanpa chart agregat)

### UI / UX
- **Dark mode** (Nuxt UI)
- **Grouping sidebar** menu
- Filter dropdown diperbaiki di semua halaman (`AppFilterSelect`, z-index popper, `AppDashboardPanel`)

### Pagination & Filter (semua tabel)
- Pagination server-side (10 item/halaman) di barang, vendor, pengguna
- Pagination client-side + filter di master data
- Komponen `AppTablePagination.vue`, util `server/utils/pagination.ts`
- Riwayat: filter jenis data + pagination di `AppHistoryTimeline`

### Export Laporan (Excel & PDF)
- Download dari navbar: barang, vendor, pengguna, riwayat
- API `GET /api/reports/{type}/export?format=xlsx|pdf`
- Library: **exceljs**, **pdfkit**
- Export mengikuti filter aktif; auth sesuai role (vendor/pengguna = HRD)
- Kolom **Harga** di laporan barang

### Barang — keterangan, evidence, harga
- Field **keterangan** (teks bebas) di barang
- **Evidence gambar** multi-upload (`BarangEvidence`, upload ke `public/uploads/barang/{id}/`)
- Komponen `AppBarangEvidenceField.vue`
- Kolom **harga** opsional (Decimal, format Rupiah di UI & laporan)
- Util format: `shared/utils/currency.ts` (`formatRupiah`, `formatCurrencyInput`, `parseCurrencyInput`)

### Riwayat perbaikan (banyak entry per barang)
- Model `BarangPerbaikan`: tanggal, deskripsi, biaya, lokasi servis, status sebelum/sesudah, catatan
- HRD tambah perbaikan langsung di detail barang
- Input **biaya** dengan format currency (`1.000.000`)
- API: `GET/POST /api/barang/[id]/perbaikan`

### Pengajuan karyawan → review HRD
- Model `BarangPengajuan`: ubah status, ganti barang, ajukan perbaikan
- Karyawan: tombol **Ajukan Perubahan** di detail barang (jika assigned)
- HRD: halaman `/pengajuan` — setujui / tolak
- API: `GET/POST /api/pengajuan`, `POST /api/pengajuan/[id]/review`

### Profil pengguna
- Halaman `/profil`: ubah nama, email, password (password lama wajib)
- API: `PUT /api/auth/profile`
- Link profil di footer sidebar

### Dokumentasi & Workflow
- Folder `checkpoint/` untuk catatan harian
- Aturan di **CURSOR.md** & **README.md**: push GitHub → wajib buat/perbarui checkpoint dulu

## Perbaikan teknis

| Masalah | Solusi |
|--------|--------|
| Filter dropdown tidak bisa diklik | `AppFilterSelect.vue`, konstanta `FILTER_ALL` / `FILTER_NONE`, z-index popper |
| Data kosong saat reload halaman | Plugin `00.session.ts`, middleware auth async, `useAuthenticatedAsyncData.ts` |
| Modal tombol hilang | Hapus pola `AppModalCard` + `#content`; pakai slot native UModal |
| Tabel terlalu panjang | Pagination + filter wajib di semua halaman daftar |
| Layout pecah saat upload evidence | Perbaikan `AppBarangEvidenceField.vue` (hidden input + grid) |
| Input harga/biaya hilang setelah reload | Format currency via ref lokal + `UInput` langsung (bukan wrapper computed) |
